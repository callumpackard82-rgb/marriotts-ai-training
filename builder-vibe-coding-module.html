Vibe-coding with Claude Code

⏱ About 18 minutes🎯 Build by describing, not typing code🔧 One real tool
1 Why this matters

Artefacts are great for one-file tools. When you want something bigger — multiple files, that reads your data, that you keep improving — you want an agentic coder: an AI that works inside your actual files, runs commands, tests its own work and fixes its own errors while you steer in plain English. People call it "vibe coding".

Claude Code is the example here. You describe what you want; it builds, runs, and shows you. You stay the architect and reviewer.

2 The idea

A tight loop you drive

Vibe coding isn't "AI does it all". It's a fast loop where you bring judgement and direction:

🗣️Describe
say what you want, plainly
⚙️It builds
writes & runs the code
👀You test
try it, read what it did
🔁Refine
"now change this"
The skills that make it work are not coding skills — they're specification (say clearly what "done" looks like), small steps (one change at a time), and review (read and test what it produced rather than trusting blindly). Keep the first project tiny.

3 Watch it work

Building a "merit points tracker" by conversation

YouBuild a simple web tool to tally house points for four houses, with + and − buttons and a reset. Single page, saved in the browser so it survives a refresh.
Claude CodeCreated index.html with four counters, buttons, and local storage. Want me to open it so you can try it?
YouYes. Also make the leading house highlighted in gold.
Claude CodeDone — the highest score now gets a gold border. Tested with sample values; it updates live. Anything else?
YouPerfect. Add a button to export the totals as a CSV.
Notice: you never saw the code unless you wanted to. You specified, it built and tested, you reviewed and refined. Small steps, each one checked. That's the whole craft.
4 Your turn

Build one tiny tool end-to-end

Ten minutes. Open Claude Code (or Cowork), start in an empty folder, and build something small and real — then refine it twice.

A first project promptIn this empty folder, build a single-file web tool: a lesson countdown timer with preset buttons (1, 5, 10 minutes), a big readable display, and a gentle sound when it finishes. No external libraries. Open it so I can test it, then wait for my feedback.

No install yet? Do the gentler version in Claude Artifacts (the rung-1 route from B1) — the describe→test→refine loop is identical; only the power differs.

5 🛡️ Data-safety checkpoint — agentic tools have real reach

It can read, run and change things — so supervise it

An agentic coder is more powerful than a chat box: it can open files, run commands and modify your machine. With that power come real responsibilities:

  • Work in a clean, dedicated folder — never point it at folders containing pupil data, safeguarding files or sensitive documents.
  • Read what it's about to do before approving commands that delete, install, or send anything. Don't rubber-stamp.
  • No secrets or pupil data in your code or test data — use synthetic examples ("House 1", "Pupil A"). Never commit API keys, passwords or personal data.
  • It's still on a managed school system — follow your acceptable-use and data-protection policies, and loop in IT.
Golden rule: you are accountable for everything the agent does in your name. Supervise it like a capable but over-eager trainee.
