1 Why this matters

Claude Code isn't the only agentic coder, and the field shifts month to month. The good news: they all work the same way, so the skill you built in B2 transfers to any of them. This module is a quick map so you can pick a lane and know what you're looking at.

Short and signposted on purpose — for Builders, the official docs are always more current than any course.

2 The idea

Same engine, different badge

Every agentic coder does the same thing: an AI that plans, writes, runs and debugs code while you steer. They differ mostly in which model powers them and where they live (terminal, IDE, cloud).

Claude Code / Cowork

Anthropic

Terminal- and app-based agent (you met it in B2). Strong all-rounder for building and refactoring.

Codex

OpenAI

OpenAI's agentic coding tool, available in the terminal, IDE and cloud.

Antigravity / Gemini CLI

Google

Google's agent-first coding IDE and command-line agent, powered by Gemini.

Copilot agent mode

Microsoft / GitHub

GitHub Copilot's agent can take a task and work across your files in the IDE.

Don't agonise over the choice. Pick the one that fits the model you already like and the place you want to work, learn it well, and know the others exist. They converge faster than they diverge.
3 Watch it work

The skills are the same in every lane

Whichever tool you choose, these three durable habits are what produce good results — they're worth more than any tool-specific trick:

📋Clear specs
say exactly what "done" looks like
🪜Small steps
one change, test, then the next
🔍Review & version control
read the diffs; use Git to undo safely
Once you're past one-file tools, learn the basics of Git — it lets you snapshot working versions and roll back when the agent breaks something. Every tool above integrates with it. It's the seatbelt that makes confident building safe.

5 🛡️ Data-safety checkpoint

Same power, same responsibilities

Everything from B2 applies to all these tools — they run with real access to your machine and sometimes the cloud:

  • No pupil data, no secrets, in any repo — especially with cloud-based agents that upload your code. Use synthetic data only.
  • Use private repositories for anything school-related; never push school code to a public repo.
  • Protect credentials and API keys — keep them out of code and out of prompts.
  • Check licensing and IT policy before installing developer tools or signing up for cloud coding services on school systems.

You remain accountable for what any agent does in your name, in every lane.
